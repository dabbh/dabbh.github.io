üö´ "Command not found" Error

Problem: The compiler/interpreter is not installed or not in PATH.

‚úÖ Solution for Python:

# Check if Python is installed
python3 --version
# or
python --version

If not installed, download from python.org

‚úÖ Solution for Java:

# Check if Java is installed
java -version
javac -version

If not installed, download from Oracle JDK or OpenJDK

‚úÖ Solution for C/C++:

# Check if GCC is installed
gcc --version
g++ --version

Windows: Install MinGW or Visual Studio
macOS: Install Xcode Command Line Tools
Linux: sudo apt install build-essential

üìÅ File Not Found Error

Problem: The file path contains spaces or special characters.

‚úÖ Solutions:

  • Avoid spaces in file names
  • Use underscores instead: my_file.py
  • Save file in a path without spaces

‚úÖ Step-by-Step Solutions

üí° Quick Tip: Most issues can be resolved by ensuring your compiler/interpreter is properly installed and accessible from the command line.

üîç Debugging Steps

  1. Check File Extension: Ensure your file has the correct extension (.py, .java, .c, etc.)
  2. Save File: Make sure the file is saved before running
  3. Check Terminal: Look at the terminal output for error messages
  4. Verify Installation: Test your compiler/interpreter in terminal

üìÅ Directory Navigation Behavior

üí° New Feature: The Run! extension automatically ensures your code runs from the correct directory.

üîí Built-in Security:

Every command is automatically prefixed with a cd to your file's directory:

# When you run: python3 script.py
# Actual execution:
Windows: cd "C:\\path\\to\\file"; if ($?) { python3 script.py }
Linux/macOS: cd "/path/to/file" && python3 script.py
